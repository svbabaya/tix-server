Документация Makefile для проекта tixerver под Axis
------------------------------
1. Инициализация и значения по умолчанию

* ARCH ?= MIPS: Устанавливает архитектуру по умолчанию. Если запустить просто make, будет выбрана ветка MIPS.
* SN ?= free: Устанавливает серийный номер по умолчанию. Если ключ не передан, программа соберется с пометкой free.
* .DEFAULT_GOAL := release_default: Определяет цель, которая выполнится при вводе команды make без аргументов. В данном случае это полная пересборка (clean) и создание release версии.

2. Блок валидации (Проверки)

* Проверка архитектуры: Сравнивает переменную ARCH со списком SUPPORTED_ARCHS (MIPS, ARMV7HF). Если введено что-то другое, make остановится с ошибкой и выведет подсказку на английском.
* Проверка SN: Если серийный номер не равен free, вызывается системная команда grep с регулярным выражением. Она проверяет, что строка состоит ровно из 12 символов и содержит только цифры 0-9 или буквы A-F. Если формат нарушен, сборка прерывается.

3. Интеграция с SDK и файловая структура

* include ... common.mak: Подключает системные настройки компилятора и пути, специфичные для Axis SDK.
* SUBDIRS & VPATH: Список папок с исходным кодом. VPATH позволяет make автоматически находить .cpp файлы в этих поддиректориях без указания полных путей в списке объектов.
* INC_FLAGS: Преобразует список папок в аргументы -Isrc -Isrc/common ... для того, чтобы компилятор видел заголовочные файлы (.h).

4. Извлечение метаданных приложения

* Парсинг package.conf: С помощью команд grep и cut из файла конфигурации достаются название приложения и части версии (Major, Minor, Micro).
* tr -d '\r': Важнейшая команда, которая удаляет символы возврата каретки. Это предотвращает ошибки компилятора (типа error: ": No such file...), если файл конфигурации редактировался в Windows.
* APP_DEFS: Формирует список макросов препроцессора. Здесь принудительно задается -DCAMERA_AXIS и передаются строки версии, имени и серийного номера (SERIAL_NUMBER) в код C++.

5. Конфигурация веток сборки (ARCH)

* Ветка MIPS:
* Указывает пути к библиотекам Libevent и OpenCV, собранным под MIPS.
   * Добавляет флаги -msoft-float -march=mips32r2 -EL для корректной генерации кода под процессор камеры.
* Ветка ARMV7HF:
* Меняет пути библиотек на build.armv7hf.
   * Макрос: Добавляет -DTraffiX_TRACKER.
   * Динамический поиск: С помощью wildcard находит все .cpp файлы в папке src/tracker и добавляет их в список сборки (OBJECTS). Также добавляет эту папку в пути поиска инклудов.

6. Флаги компиляции и линковки

* CXXFLAGS / CFLAGS: Объединяют все макросы, пути к библиотекам SDK и настройки стандартов (-std=c++11 и -std=c89).
* LDFLAGS: Указывает линковщику, где искать файлы библиотек (.so, .a). Параметр -Wl,-rpath-link необходим для разрешения перекрестных зависимостей внутри SDK.
* LDLIBS: Перечисляет конкретные библиотеки, которые нужно подключить к проекту (OpenCV, Libevent, Capture и др.).

7. Правила сборки (Targets)

* debug / release: Различаются только флагами оптимизации (-O0 против -O2) и наличием отладочных символов (-g). В release также вызывается STRIP для уменьшения размера бинарного файла.
* $(TARGET): Конечная цель. Собирает все объектные файлы (.o) в один исполняемый файл tixerver.
* %.o: %.cpp: Универсальное правило: как из любого исходника .cpp получить объектный файл .o.
* clean: Очищает рабочую директорию от результатов предыдущих сборок.
